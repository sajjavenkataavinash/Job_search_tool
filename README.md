# Job Search Tracker — Venkata Avinash Sajja

A personal job search tracker that automatically fetches PM/TPM roles daily, scores them against my profile, auto-generates tailored resumes using Claude AI, and tracks application status — all in one dashboard.

**Live Dashboard:** [sajjavenkataavinash.github.io/Job_search_tool](https://sajjavenkataavinash.github.io/Job_search_tool)

---

## What It Does

- **Daily job fetch** — GitHub Actions runs every morning at 9 AM ET, pulls PM/TPM job listings from LinkedIn, Indeed, Glassdoor, and ZipRecruiter via the JSearch API (posted in last week, US only, full-time)
- **Profile matching** — Jobs are scored against keywords from my background (GCP, AWS, Kubernetes, Terraform, API, B2B, enterprise, data platform, fintech, observability, etc.). Director/VP/Head-of roles are automatically excluded.
- **Auto resume generation** — Jobs with a match score ≥ 7 automatically get a tailored resume generated by Claude Haiku, injected into my exact resume template with correct personal details and styling
- **Manual resume generation** — Any job can have a tailored resume generated on-demand via the "Generate" button in the dashboard (requires a GitHub PAT with `workflow` scope)
- **Resume match score** — Each generated resume is scored (0–100%) for how well it matches the job description
- **Application tracking** — Track status per job: Not Applied → Applied → In Interview → Rejected (saved in browser localStorage)
- **Filters** — Search by role/company, filter by status, source, match score tier, and sort by best match or newest first
- **Datadog RUM** — Real user monitoring on the dashboard for pageviews and interactions

## How to Use

1. Open the dashboard
2. Click **Fetch New Jobs** to open GitHub Actions and manually trigger a fetch (or wait for the daily 9 AM ET run)
3. Browse matched jobs, click **View Job** to open the listing
4. Click **Generate** on any job to trigger a tailored resume — enter your GitHub PAT when prompted (stored in browser only, never in code)
5. Click **View Resume** once generated to open the tailored HTML resume
6. Update **Application Status** per row using the dropdown — saved in your browser

## Resume Generation

Resumes are generated by Claude Haiku using `generate-resume.js`:
- Uses `resume-template.html` — an exact copy of the real resume with `{{PLACEHOLDER}}` markers
- Personal details (name, email, phone, LinkedIn, location) are preserved exactly from the template
- Location is set to: `Atlanta Metropolitan Area · Open to Remote, Hybrid`
- Only tailored content (summary, highlights, bullet groups) is replaced per job
- All metrics and achievements are kept intact — only emphasis and ordering changes
- Generated resumes saved to `resumes/resume_{job_id}.html` and committed automatically

## Setup

### Required GitHub Secrets
Go to: GitHub repo → Settings → Secrets and variables → Actions → New repository secret

| Secret | Description |
|--------|-------------|
| `RAPIDAPI_KEY` | JSearch API key from rapidapi.com (subscribe to JSearch BASIC plan) |
| `ANTHROPIC_API_KEY` | Anthropic API key for Claude Haiku resume generation |

### Enable GitHub Pages
GitHub repo → Settings → Pages → Source: Deploy from branch → Branch: `main` → Folder: `/` (root)

## Stack

- Pure HTML/CSS/JS frontend (no framework)
- Node.js 20 (`fetch-jobs.js`, `generate-resume.js`) for GitHub Actions
- JSearch API (RapidAPI) — aggregates LinkedIn, Indeed, Glassdoor, ZipRecruiter
- Claude Haiku (`claude-haiku-4-5-20251001`) for resume tailoring
- GitHub Actions for daily automation and on-demand resume generation
- GitHub Pages for hosting
- `jobs.json` as the data store (auto-updated daily, max 200 jobs)
- Datadog RUM for monitoring
